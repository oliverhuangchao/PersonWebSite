56 /*
57  We define a complete-field prefix of a key value as a prefix where
58  the last included field in the prefix contains the full field, not
59  just some bytes from the start of the field. A partial-field prefix
60  is allowed to contain only a few first bytes from the last included
61  field.
62 
63  Below HA_READ_KEY_EXACT, ..., HA_READ_BEFORE_KEY can take a
64  complete-field prefix of a key value as the search
65  key. HA_READ_PREFIX and HA_READ_PREFIX_LAST could also take a
66  partial-field prefix, but currently (4.0.10) they are only used with
67  complete-field prefixes. MySQL uses a padding trick to implement
68  LIKE 'abc%' queries.
69 
70  NOTE that in InnoDB HA_READ_PREFIX_LAST will NOT work with a
71  partial-field prefix because InnoDB currently strips spaces from the
72  end of varchar fields!
73 */
74
